# Guide Index

This guide is organized for onboarding, not for completeness.

If someone is anxious about Git, the goal is to help them build confidence in this order:

1. understand what problem Git solves
2. understand the basic objects they interact with
3. learn the normal day-to-day workflow
4. learn how collaboration happens through GitHub
5. learn how to recover from mistakes

## How To Use This Guide

- Read sections `01` through `06` in order if you are new.
- Jump to `07` if you are stuck and need to recover.
- Keep `09` and `10` open as reference material.
- Treat `08` as the team's source of truth once your workflow is agreed.
