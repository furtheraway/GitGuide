# AGENTS.md

Repository-specific instructions for Codex and future editing sessions.

## Working Style

- Build this guide incrementally in small, reviewable pieces.
- Prefer editing one section or one idea at a time.
- Avoid large batch rewrites unless explicitly requested.
- Preserve the user's draft language and outline direction where possible.
- When restructuring content, explain the reason briefly before making the change.

## Source Of Truth

- `GUIDE.md` is the current consolidated handbook.
- `git-guide-self-built.md` is the step-by-step working draft for gradually building the guide in the user's own logic flow.

## Editing Preference

- Prefer adding or refining a small section instead of generating a full replacement document.
- Work out concepts sentence by sentence.
- Be very concise.
- Do not extend beyond what the user directly asks for.
- Do not add framing such as "it is helpful to think" unless the user explicitly asks for that kind of explanation.
- Do not add adjacent sections such as "what is Git used for" unless the user explicitly asks for them.
- Use plain language and practical examples over textbook-style explanation.
- Keep the guide aligned with the team's real workflow and collaboration habits.

## Git Preference

- Make small commits with clear intent when asked to commit.
- Treat prior autogenerated content as a starting point, not as fixed structure.
